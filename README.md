# AI Поиск — Android (Kotlin)

Нативное Android-приложение, повторяющее твой веб-прототип (Perplexity-style поиск),
но AI-ответы и поисковые системы грузятся в настоящих `WebView` внутри приложения,
а не открываются в браузере.

## Что реализовано

- **Вкладки браузера** (как в Chrome) — можно открывать несколько вкладок кнопкой «+»,
  закрывать крестиком, переключаться. Каждая вкладка хранит свои 3 WebView (по одному
  на под-вкладку), так что состояние не теряется при переключении.
- **Под-вкладки внутри каждой вкладки**: «Ответ» / «Ссылки» / «Изображения» — как в
  твоём веб-варианте.
- **«Ответ»**: переключатель между Gemini, ChatGPT.com и Claude.ai (кнопка
  «сменить ИИ» циклически переключает, либо можно нажать конкретный чип).
- **«Ссылки» / «Изображения»**: переключатель между Google, Bing, DuckDuckGo, Yandex
  (выпадающее меню по нажатию на текущее название + ссылка «сменить поиск» для
  цикличного переключения).
- Ссылки, открывающиеся в новом окне (`target="_blank"`), автоматически открываются в
  новой вкладке приложения.
- Системная кнопка «Назад» сначала листает историю текущего WebView, а не сразу
  закрывает приложение.

## Структура проекта

```
app/src/main/java/com/maxdev/aisearchbrowser/
  Models.kt        — enum SearchEngine, AiService, SubTab + построение URL
  BrowserTab.kt     — состояние одной вкладки (3 WebView + текущие настройки)
  MainActivity.kt   — вся UI-логика (интерфейс собран программно, без XML-layout)
```

## Как открыть и собрать

1. Открой папку `AiSearchBrowser` в Android Studio (Hedgehog/Koala или новее) как
   существующий проект — File → Open.
2. Android Studio при первом открытии сам сгенерирует `gradlew`/`gradle-wrapper.jar`
   и подтянет зависимости (для этого понадобится интернет). Если он спросит про
   Gradle Sync — соглашайся.
3. Запусти на эмуляторе или устройстве (минимальный SDK — 24, Android 7.0).

Если хочешь собрать через консоль (`gradlew assembleDebug`), сначала один раз открой
проект в Android Studio, чтобы она создала `gradle/wrapper/gradle-wrapper.jar` — я не
могу сгенерировать этот бинарный файл без доступа в сеть.

## Важные технические ограничения

- **Вход в Google-аккаунт (Gemini) может не работать.** Начиная с 2016 года Google
  намеренно блокирует OAuth-вход во встроенных WebView в целях безопасности — это
  ограничение самого Google, а не баг приложения. Если вход не проходит, единственный
  надёжный обход — использовать Chrome Custom Tabs для самого шага логина, а потом
  возвращаться в WebView с уже готовой cookie-сессией. Это можно добавить отдельным
  шагом, если понадобится.
- ChatGPT и Claude обычно работают во WebView нормально (email/пароль или сессия по
  cookie), но у ИИ-сервисов время от времени меняются структура страницы и защита от
  ботов — может понадобиться донастройка User-Agent.
- Юзер-агент по умолчанию — «desktop-like» (убран маркер `; wv`), это помогает многим
  сайтам не блокировать встроенный браузер, но не гарантирует 100% совместимость.

## Товарные знаки

Приложение обращается к публичным сайтам Google Gemini, OpenAI ChatGPT и Anthropic
Claude по их обычным URL — это не то же самое, что использование чужого кода или
логотипов. Как и с твоим веб-прототипом: не используй официальные логотипы, добавь в
README дисклеймер об отсутствии аффилиации, и не давай проекту имя, похожее на
торговую марку.
