# Сборка APK через GitHub Actions (без компьютера)

Это позволяет собрать `.apk` прямо на серверах GitHub — всё, что нужно, это браузер
и аккаунт GitHub (можно даже с телефона).

## 1. Создай репозиторий на GitHub

1. Зайди на github.com → New repository.
2. Назови его как хочешь (например `ai-search-browser`), можно **Private**.
3. НЕ добавляй README/gitignore при создании (они уже есть в проекте) — просто
   создай пустой репозиторий.

## 2. Залей туда файлы проекта

Проще всего — через веб-интерфейс GitHub, без git и без компьютера:

1. Открой свежесозданный репозиторий → кнопка **"uploading an existing file"**
   (или "Add file" → "Upload files").
2. Перетащи содержимое папки `AiSearchBrowser` (все файлы и подпапки: `app/`,
   `.github/`, `build.gradle.kts`, `settings.gradle.kts`, и т.д.).
   - Важно: загружай именно **содержимое** папки `AiSearchBrowser`, а не саму папку
     как один элемент — то есть `app`, `.github`, `build.gradle.kts` и т.п. должны
     оказаться в корне репозитория.
3. Внизу нажми **"Commit changes"**.

Если браузер не даёт загрузить папки/скрытые папки (`.github`) через drag-n-drop —
загрузи файлы `.github/workflows/android-build.yml` отдельно через "Add file" →
"Create new file" и вручную впиши путь `.github/workflows/android-build.yml`, вставив
содержимое файла.

## 3. Сборка запустится сама

Как только ты закоммитишь файлы в ветку `main`, во вкладке **Actions** твоего
репозитория автоматически запустится workflow **"Build APK"**. Сборка занимает
обычно 2–4 минуты.

Если хочешь запустить сборку вручную (без нового коммита):
1. Вкладка **Actions**.
2. Слева выбери **"Build APK"**.
3. Справа кнопка **"Run workflow"** → **"Run workflow"**.

## 4. Скачай готовый APK

1. Вкладка **Actions** → открой последний (зелёный ✓) запуск **"Build APK"**.
2. Внизу страницы раздел **"Artifacts"** → скачай **`app-debug-apk`** (это zip,
   внутри — `app-debug.apk`).
3. Перекинь `.apk` на телефон (через Telegram "Избранное", Google Drive, USB-кабель
   и т.д.) и установи (Android спросит разрешение на установку из неизвестных
   источников — это нормально для debug-сборки).

## Если сборка упадёт с ошибкой

Открой упавший запуск в Actions → шаг с крестиком покажет лог ошибки. Самые частые
причины:
- Файлы залиты не в корень репозитория (например, всё оказалось внутри
  `AiSearchBrowser/AiSearchBrowser/...`) — проверь структуру через вкладку **Code**:
  `app/` и `.github/` должны быть видны сразу в корне.
- Опечатка при ручном создании `.github/workflows/android-build.yml`.
